## Workflow Rules (Permanent)

- **Commit policy**: 이슈 브랜치에 직접 커밋. `Co-Authored-By: Claude` 또는 "클로드코드 협업자" 라인 절대 금지.
- **After implement**: 품질 게이트 통과 후 이슈 브랜치에 커밋 → GitHub PR 생성 후 URL 보고.

## 코드 품질 기준

에이전트 종류와 무관하게 코드 생성·수정 시 아래 4개 축을 기준으로 삼는다.

### 가독성

- **단일 책임**: 함수·모듈은 하나의 목적만 수행한다.
- **의도가 드러나는 이름**: 변수·함수·타입 이름만으로 역할을 알 수 있어야 한다. 약어·범용 이름(`data`, `tmp`, `x`) 지양.
- **매직 넘버·문자열 금지**: 리터럴 값은 이름 있는 상수로 추출한다.
- **짧고 집중된 함수**: 화면 한 페이지를 넘기 전에 기능 단위로 분리를 고려한다.

### 예측 가능성

- **프로젝트 관용구 우선**: 기존 코드베이스의 패턴·컨벤션을 새 코드에도 동일하게 적용한다. 불필요한 스타일 혼합을 피한다.
- **사이드이펙트 최소화**: 함수가 인자 외 외부 상태를 변경한다면 이름·주석으로 명시한다.
- **암묵적 타입 변환 금지**: TypeScript 환경에서 `as any`, 암묵적 `undefined` 반환을 피하고 명시적 타입을 사용한다.

### 응집도

- **관련 로직 한 곳에**: 동일 도메인의 로직을 여러 파일에 분산하지 않는다.
- **불필요한 추상화 지연**: 중복이 세 번 이상 발생하기 전까지 공통 유틸리티 추출을 서두르지 않는다.
- **레이어 역할 준수**: 라우트 핸들러는 비즈니스 로직을 갖지 않고, 서비스는 HTTP 컨텍스트를 알지 않는다.

### 결합도 & 기본 성능

- **의존성 단방향**: 모듈 의존성 방향을 단방향으로 유지한다. 순환 의존성을 도입하지 않는다.
- **N+1 쿼리 금지**: 루프 내 DB 쿼리는 배치 쿼리 또는 join으로 대체한다.
- **무한 루프·메모리 누수 주의**: 종료 조건 없는 재귀·루프, 해제되지 않는 이벤트 리스너·구독을 만들지 않는다.
- **불필요한 렌더링 방지**: React 컴포넌트에서 매 렌더마다 새 객체·함수를 생성하는 패턴을 피한다.
